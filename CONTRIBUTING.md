# Руководство по внесению вклада

## Как внести вклад

1. Fork репозитория
2. Создайте ветку для вашей функции (`git checkout -b feature/AmazingFeature`)
3. Commit ваши изменения (`git commit -m 'Add some AmazingFeature'`)
4. Push в ветку (`git push origin feature/AmazingFeature`)
5. Откройте Pull Request

## Стандарты кода

### Python (Backend)
- Следуйте PEP 8
- Используйте type hints
- Документируйте функции docstrings
- Максимальная длина строки: 100 символов

### TypeScript/React (Frontend)
- Используйте ESLint и Prettier
- Компоненты должны быть функциональными
- Используйте TypeScript для всех новых файлов
- Следуйте правилам именования: PascalCase для компонентов, camelCase для функций

## Тестирование

Перед отправкой PR убедитесь, что:
- Все тесты проходят
- Код соответствует стандартам (lint)
- Документация обновлена

## Вопросы

Если у вас есть вопросы, откройте Issue в репозитории.
